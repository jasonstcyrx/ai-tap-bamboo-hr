Notes You may now specify a special employee ID of "all", which will cause the API to return all rows for all employees in the table. When specifying "all" employees, the result will include inactive and terminated employees. As with all other API methods, the list of employees shown is still filtered based on the permissions of the user that makes the API request. To get a list of all table data for all employees, make sure to use the API key of a user that has permission to view all the data.
